- (decision) Paradigm: Feature-Sliced Design (FSD) with hexagonal core — business logic in entities/features isolated from framework concerns (Next.js pages, DB adapters). Prevents server actions leaking into business rules or entities depending on HTTP.
- (decision) AD-1: Server Actions as mutation boundary — all data writes go through Server Actions (never direct DB from client). Prevents client-side data tampering.
- (decision) AD-2: Route Handlers for external API proxies only — exchange rate fetches and export generation. Never for business CRUD.
- (decision) AD-3: Supabase RLS enforced as auth gate on all tables — users access only their own rows (user_id = auth.uid()). Prevents horizontal data access leaks.
- (decision) AD-4: Zod schemas as single source of truth for validation — shared between client forms and server actions. Prevents validation drift.
- (decision) AD-5: TanStack Query for server state, React Context only for UI state (theme, sidebar). Prevents over-fetching and stale data.
- (decision) AD-6: VAT engine as pure service function — tax calculation isolated in shared/lib with zero side effects. Takes amount + rate, returns tax. Prevents tax logic leaking into components.
- (decision) AD-7: Exchange rate service with cache layer — fetches from Frankfurter API, caches in DB/Redis, falls back to stale cache on failure. Prevents API dependency becoming a hard failure.
- (decision) AD-8: Next.js App Router — Server Components by default, client components only where interactivity is required. Maximizes server-side rendering and reduces client JS bundle.
- (decision) AD-9: Shared/lib for business logic — VAT engine, currency converter, date formatting, number formatting. No business logic in app/ or widgets/.
- (decision) AD-10: Entity layer owns all DB access — repositories in entities/expenses, entities/categories, entities/settings encapsulate Supabase queries. Prevents raw DB calls scattered across features.
- (event) Architecture spine drafted with 10 ADs covering mutation boundary, API proxy rule, RLS isolation, Zod validation, TanStack Query, VAT engine, currency cache, Server Components, business logic encapsulation, and entity repositories.
